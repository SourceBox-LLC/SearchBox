### Internal Use (No AGPL Trigger)

- ✅ Running SearchBox within your company
- ✅ Employees accessing the system
- ✅ Internal document search
- ❌ **No source disclosure required**

### External Use (AGPL Applies)

- ⚠️ Public-facing search service
- ⚠️ Offering SearchBox as SaaS
- ⚠️ Customer-facing deployment
- ✅ **Must provide source code to users**

## AGPL vs MIT vs Apache

| Aspect | AGPL | MIT | Apache 2.0 |
|--------|------|-----|------------|
| **Can use in closed-source?** | ❌ No | ✅ Yes | ✅ Yes |
| **Can run as service without source?** | ❌ No | ✅ Yes | ✅ Yes |
| **Can re-license?** | ❌ No | ✅ Yes | ✅ Yes |
| **Patent grant?** | ❌ No | ❌ No | ✅ Yes |
| **SaaS loophole closed?** | ✅ Yes | ❌ No | ❌ No |

**Why AGPL?** Protects open-source project from:
- Companies taking code without contributing back
- SaaS providers offering closed-source versions
- Commercial products without source disclosure

## FAQ

**Q: Can I use SearchBox at my company?**

A: Yes! Internal use is free with no requirements. No source disclosure needed for internal use.

**Q: Do I need to share my modifications?**

A: Only if you:
- Distribute modified versions publicly, OR
- Run modified SearchBox as a public web service

**Q: Can I charge for hosting SearchBox?**

A: Yes, but you must:
- Provide source code access to users
- Use AGPL license for your hosting
- Include original license and notices

**Q: What if I want to keep modifications private?**

A: Purchase a commercial license — see above.

**Q: Can I use SearchBox code in my project?**

A: Yes, but your project must also be AGPL-licensed.

**Q: Does AGPL affect my other code?**

A: Only code that links to or incorporates SearchBox is affected. Seperate processes are fine.
