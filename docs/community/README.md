## Getting Help

### GitHub Discussions

For questions and help:

1. Go to [GitHub Discussions](https://github.com/SourceBox-LLC/SearchBox/discussions)
2. Search for your question first
3. If not found, create a new discussion:
   - **Q&A** — Questions about usage
   - **Ideas** — Feature suggestions
   - **Show & Tell** — Share your projects

### Best Practices for Questions

**Good question:**
> "I'm getting 'Connection refused' when starting Meilisearch. I've checked that port 7700 is not in use. What else could cause this?"

**Not so good:**
> "Meilisearch won't start. Help?"

### Before Asking

1. **Search documentation** — Your answer may already be there
2. **Check existing issues** — Someone may have reported it
3. **Gather information:**
   - SearchBox version
   - Python version
   - Operating system
   - Error messages
   - Steps to reproduce

## Reporting Bugs

### Before Reporting

1. **Search existing issues** — Avoid duplicates
2. **Try the latest version** — Bug may be fixed
3. **Gather info:**
   - Steps to reproduce
   - Expected behavior
   - Actual behavior
   - Logs/screenshots

### Bug Report Template

```markdown
**Description:**
Brief description of the bug.

**Steps to Reproduce:**
1. Go to '...'
2. Click on '...'
3. See error

**Expected Behavior:**
What should happen.

**Actual Behavior:**
What actually happens.

**Environment:**
- SearchBox version: [e.g., 1.0.0]
- Python version: [e.g., 3.10]
- OS: [e.g., Ubuntu 22.04]
- Deployment: [e.g., Docker, native]

**Logs:**
```
Paste relevant logs here
```

**Screenshots:**
If applicable.
```

## Feature Requests

### Before Requesting

1. **Search discussions** — May be under consideration
2. **Check roadmap** — May be planned
3. **Consider scope** — Is it a small improvement or major feature?

### Feature Request Template

```markdown
**Is your feature request related to a problem?**
A clear description of the problem.

**Describe the solution you'd like:**
What you'd like to see implemented.

**Describe alternatives you've considered:**
Other solutions you've thought of.

**Use case:**
How would this feature be used?

**Priority:**
Low / Medium / High
```

## Code of Conduct

### Our Pledge

We are committed to providing a welcoming and inspiring community for all.

### Standards

**Positive behavior includes:**
- Being respectful and inclusive
- Showing empathy towards others
- Accepting constructive criticism
- Focusing on what's best for the community

**Unacceptable behavior includes:**
- Harassment or discrimination
- Trolling or insulting comments
- Public or private harassment
- Publishing others' private information

### Enforcement

Violations may result in:
- Warning
- Temporary ban
- Permanent ban

Report violations to: conduct@sourcebox.dev

## Security

Found a security vulnerability? **Do not open a public issue.**

Email: security@sourcebox.dev

We follow responsible disclosure:
1. Report privately via email
2. We acknowledge within 48 hours
3. We investigate and fix
4. We coordinate public disclosure

See [Security Policy](../architecture/security.md) for details.
